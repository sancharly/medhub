# TASK-080 — Typed API client + cookie/CSRF + TanStack Query

- **Phase:** 6 — Frontend foundation
- **Software item / unit:** SI-FE-APICLIENT / U-FEAPI-Client
- **Implements:** SR-031 (AC-3 CSRF token on state-changing requests; AC-5 server-side authz is authoritative, client never the sole gate); NFR-006 (typed interface generated from the OpenAPI contract); ADR-0003, ADR-0012
- **Depends on:** TASK-068 (OpenAPI 3.1 contract + typed-client generation artifact) — must be merged first
- **Branch:** `enabler-story/fe-api-client`
- **Status:** In Progress (audit 2026-06-29)

## Objective

Stand up the **single typed gateway** every frontend item uses to reach the backend: a client whose method signatures and DTO types are **generated from the OpenAPI 3.1 document** produced by TASK-068 (NFR-006 typed interfaces), wrapped so that the opaque session cookie rides automatically and a CSRF token is attached to every state-changing request (SR-031.3). It also installs the TanStack Query client/provider that all data-fetching views (Phase 7/8) build on. This unit is the only place the SPA configures transport, credentials, and CSRF — no feature view sets headers itself. Traces to SR-031, NFR-006.

## Interfaces to honor

Provides the typed `ApiClient` from [12-interfaces.md](../../design/12-interfaces.md) §12.2 (provider SI-FE-APICLIENT; consumers SI-FE-AUTH, SI-FE-CORE, modules):

```ts
// methods + DTO types are GENERATED from /api/v1/openapi.json (TASK-068), not hand-written
// every method attaches the session cookie + CSRF token (SR-031.3)
const me = await apiClient.getMe();                 // GET  /me
await apiClient.login({ email, password });          // POST /auth/login  (state-changing → CSRF)
await apiClient.logout();                            // POST /auth/logout (state-changing → CSRF)
```

- Consumes the **generated** client + DTO types from the TASK-068 artifact (`/api/v1/openapi.json`); types are not hand-authored, so the SPA shares the backend contract end-to-end (NFR-006, ADR-0003).
- Transport sends the cookie on every request (`fetch` `credentials: 'include'`); the cookie is `Secure; HttpOnly; SameSite=Strict` and set/cleared **only** by the backend (ADR-0012) — the client never reads or writes it.
- On state-changing verbs (POST, PUT, PATCH, DELETE) the client attaches the CSRF token in the `X-CSRF-Token` header, read from the non-HttpOnly CSRF cookie issued at login (TASK-022 / SR-031.3).
- Must **not**: enforce or assume authorization client-side as the sole control — server-side authz is authoritative (SR-031.5, ADR-0003 negative consequence); store the session token in JS-accessible storage (`localStorage`/`sessionStorage`); hand-edit generated DTO types; swallow RFC 7807 error bodies (it surfaces them as typed errors for TASK-084).

## Implementation detail

- Files to create:
  - `frontend/src/api/generated/` — output of the TASK-068 generator (client methods + DTO types) consumed from `/api/v1/openapi.json`; checked in and regenerated by the `generate:api` script (TASK-068). Treated as generated — never hand-edited.
  - `frontend/src/api/client.ts` — the `ApiClient` wrapper around the generated client: configures base URL `/api/v1`, `credentials: 'include'`, a request interceptor that injects `X-CSRF-Token` from the CSRF cookie on state-changing methods, and a response path that parses `application/problem+json` into a typed `ProblemError` (consumed by TASK-084).
  - `frontend/src/api/csrf.ts` — read the CSRF token from its cookie; helper `withCsrf(headers)`.
  - `frontend/src/api/queryClient.ts` — the TanStack Query `QueryClient` (sane defaults: no refetch-on-window-focus for clinical data, retry only on network errors not on 4xx).
  - `frontend/src/api/QueryProvider.tsx` — `QueryClientProvider` wrapper mounted at the app root.
  - `frontend/src/api/index.ts` — exports the singleton `apiClient`, `queryClient`, `ProblemError`.
- Libraries (exact stack): TypeScript, React 18, TanStack Query, `fetch` (standards-based, ADR-0003 cross-browser), the OpenAPI client generator wired in TASK-068.
- CSRF: the token is delivered in a readable cookie at login (TASK-022); the client copies it into `X-CSRF-Token` for unsafe methods. Safe methods (GET/HEAD) send no CSRF header. A missing token on a state-changing call is allowed to reach the server, which returns `403 /errors/forbidden` (TASK-069) — the client does not invent tokens.
- Error mapping: any non-2xx with `Content-Type: application/problem+json` is parsed into `ProblemError { type, title, status, detail, errors? }` and thrown so TanStack Query exposes it to views; `401 /errors/unauthenticated` is surfaced as a typed auth error the shell auth boundary (TASK-085) reacts to (redirect to login).
- This unit performs no rendering and owns no routes; it is pure transport + query infrastructure.

## Tests (write first — TDD, CLAUDE.md §4)

- Unit tests (Vitest + MSW), `frontend/src/api/client.test.ts`:
  - a GET (`getMe`) is issued with `credentials: 'include'` and **no** `X-CSRF-Token` header (SR-031.3 — safe method).
  - a POST/PUT/PATCH/DELETE attaches `X-CSRF-Token` read from the CSRF cookie (SR-031.3).
  - a `4xx`/`5xx` returning `application/problem+json` is parsed into a typed `ProblemError` carrying `type`/`detail`/`errors[]` (RFC 7807, for TASK-084).
  - a `401 /errors/unauthenticated` is surfaced as the typed auth error (consumed by the shell boundary, SR-031.5 — client is not the sole gate).
  - the session cookie is never read from `document.cookie`/`localStorage` by the client (ADR-0012 — HttpOnly).
- Provider test (Vitest + RTL), `frontend/src/api/QueryProvider.test.tsx`: a child `useQuery` hook resolves through the provider against an MSW-mocked endpoint.
- Each test names the SR/NFR it verifies.

## Acceptance criteria

- [ ] Client methods and DTO types are generated from `/api/v1/openapi.json` (TASK-068), not hand-written (NFR-006).
- [ ] Every request sends the session cookie via `credentials: 'include'`; the client never reads/writes the HttpOnly session cookie (ADR-0012).
- [ ] State-changing requests (POST/PUT/PATCH/DELETE) attach `X-CSRF-Token`; safe requests do not (SR-031.3).
- [ ] RFC 7807 `application/problem+json` responses are parsed into a typed error for the error surface (SR-027.3 boundary, consumed by TASK-084).
- [ ] Authorization is never enforced client-side as the sole control; the client defers to server decisions (SR-031.5).
- [ ] A TanStack Query `QueryClient` + provider is available to all feature views.
